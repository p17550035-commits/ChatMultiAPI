+----------------------------------------------------------+
| SECTION 1 — PURPOSE                                      |
+----------------------------------------------------------+
| The heartbeat subsystem provides periodic communication   |
| between every module and its router. It enables:          |
| - Real-time health monitoring                             |
| - Automatic failure detection                             |
| - Automatic redundancy switching                          |
| - Automatic recovery attempts                             |
| - Automatic notification triggers                         |
| - Full logging for debugging and forensic analysis        |
+----------------------------------------------------------+
| SECTION 2 — HEARTBEAT PACKET STRUCTURE                   |
+----------------------------------------------------------+
| HeartbeatPacket:                                         |
|   module_id: string                                      |
|   timestamp: int (unix epoch)                            |
|   status: enum(OK, DEGRADED, OFFLINE)                    |
|   last_error: string|null                                |
|   uptime: int                                            |
|   load: float|null                                       |
|   correlation_id: string (UUID)                          |
|   router_log_pointer: string                             |
|   hub_log_pointer: string                                |
| Routers forward packets to:                              |
|   - LayerHub                                              |
|   - MasterHub (primary)                                   |
|   - RedundantHub (secondary)                              |
+----------------------------------------------------------+
| SECTION 3 — LOGGING SYSTEM                               |
+----------------------------------------------------------+
| RouterLog:                                               |
|   event_type: HEARTBEAT_RECEIVED, HEARTBEAT_MISSED,      |
|               VALIDATION_ERROR, ROUTE_DECISION           |
|   module_id: string                                      |
|   timestamp: int                                         |
|   correlation_id: string                                 |
|   details: object                                        |
| Stored at: /logs/router/<module>/<date>/<id>.log         |
|                                                          |
| HubLog:                                                  |
|   event_type: STATUS_CHANGE, REDUNDANCY_SWITCH,          |
|               RECONNECT_ATTEMPT, RECOVERY_SUCCESS,       |
|               RECOVERY_FAILURE, NOTIFICATION_TRIGGER     |
|   layer_id: string                                       |
|   timestamp: int                                         |
|   correlation_id: string                                 |
|   details: object                                        |
| Stored at: /logs/hub/<layer>/<date>/<id>.log             |
|                                                          |
| MasterHubLog:                                            |
|   event_type: GLOBAL_MAP_UPDATE, BREAKER_SWITCH,         |
|               ORCHESTRATION_FAILURE, ORCHESTRATION_RECOV |
| Stored at: /logs/master/<date>/<id>.log                  |
|                                                          |
| RedundantHubLog:                                         |
|   event_type: TAKEOVER, PING, RECOVERY_DETECTED,         |
|               FALLBACK_ROUTING                           |
| Stored at: /logs/redundant/<date>/<id>.log               |
+----------------------------------------------------------+
| SECTION 4 — FAILURE DETECTION LOGIC                      |
+----------------------------------------------------------+
| If module misses X heartbeats:                           |
|   Router logs HEARTBEAT_MISSED                           |
|   Router sets status = DEGRADED                          |
|   Hub logs STATUS_CHANGE                                 |
|   Hub sets status = OFFLINE                              |
|   Hub triggers notifications + redundancy + calendar     |
|
